You are reviewing extraction proposals from THREE parallel passes that examined the same conversation. Each pass focused on a different memory axis. Your job is to identify cross-axis duplicates and drop the weaker version.

## Pass A — UserProfile (user-side facts and preferences)
{pass_a_ops}

## Pass B — UserProfileAgentRec (agent-named answers / recommendations the agent gave the user)
{pass_b_ops}

## Pass C — UserPlaybook (procedural rules / playbooks for the agent to follow)
{pass_c_ops}

## Rules

1. **Same fact captured twice → drop the weaker one.**
   - If Pass B captured an "agent recommendation" that is really just restating a Pass A user fact, drop the Pass B op.
   - If Pass A captured a fact that already covers the substance of a Pass C playbook trigger, drop only when the playbook adds NO operational guidance.
2. **Different angles on the same event are NOT duplicates.** Keep both.
   - Example: Pass A "user told assistant they are vegan" + Pass B "agent recommended a vegan restaurant" — KEEP BOTH (one is the user's fact, the other is the agent's answer).
3. **Never drop DELETE ops.** They reflect storage-state reasoning the agent did during its loop.
4. **Never drop ops that contain unique operands** (specific numbers, dates, names, prices, exact phrases) — those are the answer-bearing details and must survive.
5. **When in doubt, KEEP.** False-keeps are recoverable; false-drops permanently lose information.

## Output format

Reply with one decision per line, in the form:

```
DROP A.<index>
DROP B.<index>
DROP C.<index>
```

If nothing should be dropped, reply with exactly: `KEEP ALL`

Use only the indices listed in the pass blocks above. Indices are 0-based and per-pass.
